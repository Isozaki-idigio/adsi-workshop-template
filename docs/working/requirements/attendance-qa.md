# 勤怠管理アプリ — 要求仕様 Q&A

## 1. 利用者・規模

[Question] このアプリの利用者は誰ですか？（例: 社員のみ / 社員＋管理者 / 社員＋管理者＋人事）
[Answer]社員＋管理者

[Question] 想定する利用人数の規模はどのくらいですか？（例: 〜50人 / 〜500人 / 1000人以上）
[Answer]500人

[Question] 管理者（上長・人事）が行う操作はありますか？（例: 部下の勤怠確認、承認、修正）
[Answer]承認、修正

## 2. 打刻

[Question] 打刻の種類は何がありますか？（例: 出勤・退勤のみ / 出勤・退勤・休憩開始・休憩終了）
[Answer]出勤、退勤のみ

[Question] 打刻方法はどのような形式ですか？（例: Web画面でボタン押下 / ICカード / GPS付きモバイル）
[Answer]web画面でボタン押下

[Question] 打刻忘れ・打刻ミスの修正フローは必要ですか？（例: 本人が申請→上長承認）
[Answer]本人が申請→上長承認

[Question] 同日に複数回の出退勤（中抜け）は許容しますか？
[Answer]許容します。

## 3. 勤務体系

[Question] 勤務形態はどれですか？（例: 固定時間制 / フレックス / シフト制 / 複数混在）
[Answer]フルフレックス

[Question] 所定勤務時間は何時間ですか？（例: 8時間 / 7.5時間）
[Answer]7.25時間

[Question] 残業の計算ルールはありますか？（例: 所定超過分を残業とする / 深夜割増あり）
[Answer]所定超過分を残業とする / 深夜割増（22:00以降）あり

## 4. 休暇・休日

[Question] 休暇管理は必要ですか？（例: 有給申請・残日数管理 / 不要）
[Answer]有給申請・残日数管理 

[Question] 休暇の種類は何がありますか？（例: 有給 / 半休 / 特別休暇 / 代休）
[Answer]有給 / 半休 / 特別休暇

[Question] 休暇申請の承認フローは必要ですか？
[Answer]必要（申請→承認→反映）

## 5. 集計・レポート

[Question] 月次の勤怠集計は必要ですか？（例: 月の労働時間合計、残業時間、遅刻回数など）
[Answer]労働時間合計のみ集計

[Question] CSV / Excel 等のエクスポート機能は必要ですか？
[Answer]csv,pdf出力

[Question] 給与計算システムとの連携は必要ですか？
[Answer]不要

## 6. 技術・非機能

[Question] 認証方式は何を使いますか？（例: ID/パスワード / SSO / OAuth）
[Answer]ID/パスワード

[Question] モバイル対応は必要ですか？（例: レスポンシブWeb / ネイティブアプリ / PC のみ）
[Answer]ネイティブアプリ（不可能であればwebアプリで）

[Question] オフライン時の打刻対応は必要ですか？
[Answer]不要

## 7. スコープ（MVP）

[Question] 最初のリリース（MVP）に含めたい機能の優先度を教えてください。以下から必須/あれば良い/不要を選んでください:
- 出退勤打刻:
- 勤怠一覧表示:
- 打刻修正申請:
- 休暇申請:
- 月次集計:
- CSV出力:
- 管理者画面:
[Answer]- 出退勤打刻:必須
- 勤怠一覧表示:必須
- 打刻修正申請:あれば良い
- 休暇申請:必須
- 月次集計:あれば良い
- CSV出力:あれば良い
- 管理者画面:必須
