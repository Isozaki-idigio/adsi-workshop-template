# 勤怠管理アプリ — 追加 Q&A

回答いただいた内容をもとに、追加で確認が必要な点です。

## モバイル対応

[Question] 本プロジェクトの技術スタックは Java Spring Boot（バックエンド）+ Next.js（フロントエンド）を想定しています。ネイティブアプリ開発（iOS/Android）は別途フレームワークが必要になりますが、レスポンシブ対応のWebアプリ（PWA含む）で代替して問題ありませんか？
[Answer] 代替OK

## 管理者の構成

[Question] 管理者は「部門ごとの上長」ですか？それとも「全社共通の管理者」ですか？（部門ごとの場合、部門マスタが必要になります）
[Answer] 部門ごとの上長

[Question] 管理者は自部門の社員の勤怠のみ見える想定ですか？それとも全社員の勤怠が見える想定ですか？
[Answer] 自部門のみ

## 打刻（中抜け）

[Question] 中抜けを許容する場合、1日の勤務時間は「全ての出勤〜退勤の合計」として計算しますか？（例: 9:00出勤→12:00退勤、14:00出勤→18:00退勤 = 計7時間）
[Answer] 合計で計算

## 休暇

[Question] 特別休暇の種類を教えてください（例: 慶弔休暇 / リフレッシュ休暇 / 育児休暇 など）。それとも管理者がマスタで自由に登録できる形式ですか？
[Answer] 固定（種類はシステムで定義）

[Question] 有給の付与ルールはどうしますか？（例: 入社日基準で自動付与 / 管理者が手動設定 / 一律4月1日に付与）
[Answer] 年間12日付与

## 承認フロー

[Question] 打刻修正と休暇申請の承認者は同一人物（直属の上長）ですか？
[Answer] 同じ人（直属の上長）

[Question] 承認者が不在の場合の代理承認は必要ですか？
[Answer] 不要
