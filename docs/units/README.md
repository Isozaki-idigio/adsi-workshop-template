# Unit of Work 分割

## 依存図

```
                    ┌─────────────────────┐
                    │  Unit 0: 共通基盤    │
                    │  (Phase A)          │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
              ▼                ▼                ▼
┌─────────────────┐  ┌─────────────────┐  ┌──────────────────────┐
│ Unit 1: 認証     │  │ Unit 2: 打刻    │  │ Unit 3: 休暇申請      │
│ (Phase B)       │  │ (Phase B)       │  │ (Phase B)            │
└────────┬────────┘  └────────┬────────┘  └──────────┬───────────┘
         │                    │                      │
         │                    ▼                      │
         │           ┌─────────────────┐             │
         │           │ Unit 4: 管理者   │             │
         └──────────▶│ (Phase C)       │◀────────────┘
                     └────────┬────────┘
                              │
                              ▼
                     ┌─────────────────┐
                     │ Unit 5: 集計    │
                     │ (Phase C)       │
                     └─────────────────┘
```

## Phase 割り当て

| Phase | Unit | 担当 | 前提 |
|-------|------|------|------|
| A | Unit 0: 共通基盤 | 合同 | なし |
| B | Unit 1: 認証 | 担当者A | Unit 0 |
| B | Unit 2: 打刻 | 担当者B | Unit 0 |
| B | Unit 3: 休暇申請 | 担当者A | Unit 0 |
| C | Unit 4: 管理者 | 担当者B | Unit 1, 2, 3 |
| C | Unit 5: 集計・エクスポート | 担当者A | Unit 2 |

## 並行実装プラン（2人）

```
時間軸 →

担当者A: [Unit 0] → [Unit 1: 認証] → [Unit 3: 休暇] → [Unit 5: 集計]
担当者B: [Unit 0] → [Unit 2: 打刻]                  → [Unit 4: 管理者]
                                                         ↑
                                              Unit 1,2,3 完了待ち
```

- Phase A（Unit 0）はペアプロまたは一方が作成してもう一方がレビュー
- Phase B で 2人が独立に実装（Unit 1+3 と Unit 2）
- Phase C で統合（管理者画面は全Unitに依存するため最後）

## MVP 対応

| Unit | MVP | 備考 |

| Unit 3 | o | 必須 |
