# 勤怠管理システム — 要求仕様書

## 概要

社員の出退勤打刻、勤怠管理、休暇申請を一元管理するWebアプリケーション。
社員と部門管理者（上長）が利用し、打刻・申請・承認のワークフローを提供する。

## 技術スタック

- バックエンド: Java / Spring Boot
- フロントエンド: Next.js（TypeScript）
- レスポンシブ対応Webアプリ（モバイル含む）

## 利用者・ロール

| ロール | 説明 |
|--------|------|
| 社員 | 自身の打刻、勤怠確認、休暇申請を行う |
| 管理者（部門上長） | 自部門の社員の勤怠確認、打刻修正・休暇申請の承認を行う |

- 想定利用人数: 約500人
- 管理者は部門ごとに設定（部門マスタが必要）
- 管理者の閲覧範囲: 自部門の社員のみ

## ユーザーストーリー

### 打刻

| ID | ストーリー | 受け入れ基準 |
|----|-----------|-------------|
| US-001 | 社員として、Web画面のボタンで出勤を打刻したい。勤務開始を記録するために。 | ボタン押下で現在時刻が出勤時刻として記録される |
| US-002 | 社員として、Web画面のボタンで退勤を打刻したい。勤務終了を記録するために。 | ボタン押下で現在時刻が退勤時刻として記録される |
| US-003 | 社員として、同日に複数回の出退勤（中抜け）を記録したい。フレックス勤務に対応するために。 | 同日に複数の出勤・退勤ペアを登録できる |
| US-004 | 社員として、打刻忘れ・ミスの修正を申請したい。正しい勤怠を記録するために。 | 修正申請を提出でき、承認後に反映される |
| US-005 | 管理者として、部下の打刻修正申請を承認/却下したい。正確な勤怠を担保するために。 | 申請一覧から承認/却下でき、承認後に勤怠に反映される |

### 勤怠管理

| ID | ストーリー | 受け入れ基準 |
|----|-----------|-------------|
| US-006 | 社員として、自分の勤怠一覧を確認したい。出退勤状況を把握するために。 | 日別の出退勤時刻・勤務時間が一覧表示される |
| US-007 | 管理者として、自部門の社員の勤怠一覧を確認したい。勤務状況を管理するために。 | 部下の勤怠を一覧で閲覧できる |
| US-008 | 管理者として、部下の勤怠を修正したい。打刻漏れ等に対応するために。 | 管理者が直接勤怠を修正できる |

### 休暇

| ID | ストーリー | 受け入れ基準 |
|----|-----------|-------------|
| US-009 | 社員として、休暇（有給/半休/特別休暇）を申請したい。休暇を取得するために。 | 種類・日付を指定して申請できる |
| US-010 | 社員として、有給の残日数を確認したい。取得計画を立てるために。 | 残日数がリアルタイムで確認できる |
| US-011 | 管理者として、部下の休暇申請を承認/却下したい。チームの人員を管理するために。 | 申請一覧から承認/却下でき、承認後に残日数が更新される |

### 集計・レポート

| ID | ストーリー | 受け入れ基準 |
|----|-----------|-------------|
| US-012 | 社員として、月次の労働時間合計を確認したい。自分の勤務状況を把握するために。 | 月ごとの労働時間合計が表示される |
| US-013 | 管理者として、部門の勤怠データをCSV/PDFで出力したい。報告資料を作成するために。 | 指定期間の勤怠データをCSV/PDFでダウンロードできる |

### 認証

| ID | ストーリー | 受け入れ基準 |
|----|-----------|-------------|
| US-014 | 利用者として、ID/パスワードでログインしたい。安全にシステムを利用するために。 | 正しいID/パスワードでログインでき、不正な場合はエラーになる |

## 勤務ルール

### 勤務形態

- **フルフレックス制**（コアタイムなし）
- 所定勤務時間: **7.25時間/日**
- 中抜け可（1日の勤務時間 = 全出退勤ペアの合計）

### 残業計算

- 所定勤務時間（7.25h）超過分 = 残業時間
- 深夜割増: 22:00〜翌5:00 の勤務に適用

### 休暇

| 種類 | 説明 |
|------|------|
| 有給休暇 | 年間12日付与。半休（0.5日）取得可 |
| 半休 | 有給の半日消化 |
| 特別休暇 | システムで固定定義（慶弔等） |

### 承認フロー

```
社員が申請 → 直属の上長が承認/却下 → 反映
```

- 打刻修正・休暇申請ともに同一の承認者（直属の上長）
- 代理承認: 不要

| 想定規模 | 約500ユーザー |
| 給与連携 | 不要 |

## 用語集

| 用語 | 定義 |
|------|------|
| 打刻 | 出勤または退勤の時刻を記録する行為 |
| 中抜け | 1日の中で一度退勤し、再度出勤すること |
| 所定勤務時間 | 1日あたりの基準勤務時間（7.25時間） |
| 残業 | 所定勤務時間を超えた労働時間 |
| 深夜割増 | 22:00〜翌5:00 の勤務に対する割増区分 |
